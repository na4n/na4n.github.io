Browser Storage

2.8.24

There are four main options for a website's client side data storage: IndexedDB, Local Storage, Session Storage, Cookies.

IndexedDB stores serialized Javascript objects, similar to passing an object through JSON.stringify(), as key value pairs in indexes. A database connection must be established and all operations done through transactions, the connection is handled asynchronously. As it stands, the API for IndexedDB is a pain point for developers, but the library idb resolves some issues. IndexedDB has the largest storage capacity of all storage methods. One interesting use case of IndexedDB is Youtube Premium which allows downloading videos to the browser; the videos are downloaded as blobs (collection of binary data AFAIK) which are read from the database when offline.

Local Storage and Session Storage are both subsets of Web Storage, where key value pairs are stored and accessed as strings. There is no specification for user-agents defining the size limit of Web Storage, however Chrome, Firefox, and Safari all allow ~5.2MB for each type. Access to Web Storage is a blocking operation, meaning that a program will stall when getting or setting sufficiently large data. Local Storage is storage that persists across sessions, whereas session storage is stored then reset when the realm (session, world, web page, ‘area’ where a Javascript program runs) is closed. The Web Storage API is quite simple, [local/session]Storage.getItem(key) and [local/session]Storage.setItem(key, value). An interesting piece of history is that Opera (before they pandered to gamers) only allowed 3MB of Web Storage.

Cookies are plaintext data that are stored and are persistent across sessions. Browsers typically support at least 300 cookies total, at least 4096 bytes per cookie, and at least 20 cookies per unique host/domain (RFC 2019 #6.3). Unlike IndexedDB and Web Storage, Cookies do not have an API for setting cookie values. The cookie cookie string (a plaintext string) is accessed with document.cookie, and which then must be parsed to add, view, or delete cookies. Cookies are persistent but expire after 400 days.
